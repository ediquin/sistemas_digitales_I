2. Postulados o Teoremas Básicos del Álgebra de Boole

El álgebra de Boole se fundamenta en un conjunto de postulados que definen las operaciones básicas sobre variables binarias. Estos postulados son la base teórica para el diseño y análisis de circuitos digitales.

N.º Nombre Intersección (AND) Unión (OR)
1 Propiedad conmutativa x·y = y·x x + y = y + x
2 Propiedad asociativa x·(y·z) = (x·y)·z x+(y+z) = (x+y)+z
3 Propiedad distributiva x·(y+z) = x·y + x·z x+(y·z) = (x+y)·(x+z)
4 Reglas de complemento x·x̄ = 0 x + x̄ = 1
5 Reglas de potencia x·x = x x + x = x
6 Reglas de absorción x·(x+y) = x x + x·y = x
7 Leyes de De Morgan (x·y)̄ = x̄ + ȳ (x + y)̄ = x̄·ȳ
8 Reglas del 0 y 1 x·1 = x, x·0 = 0 x+1 = 1, x+0 = x
9 Involución (x̄)̄ = x

3. Demostraciones con Tablas de Verdad

Las tablas de verdad nos permiten verificar la validez de los postulados del álgebra de Boole de manera sistemática. Cada combinación posible de variables de entrada se evalúa para demostrar que ambos lados de una ecuación producen los mismos resultados.

Ejemplo: Distribución del AND sobre el OR

Demostrar: x·(y+z) = x·y + x·z

✅ Coinciden todas las salidas ⇒ Propiedad demostrada

Como podemos observar, las columnas x·(y+z) y x·y + x·z tienen valores idénticos en todas las filas, lo que confirma matemáticamente la validez de la propiedad distributiva.

4. Minitérminos y Maxitérminos

📌 ¿Qué son?

Los minitérminos y maxitérminos son formas canónicas de representar funciones booleanas que nos permiten expresar cualquier función lógica de manera estándar y sistemática.

Minitérminos (m): Combinaciones donde la función es 1, usando AND de variables y sus negaciones.
Se representa como: mi = A·B̄·C (según la fila que representa)
Maxitérminos (M): Combinaciones donde la función es 0, usando OR de variables y sus negaciones.
Se representa como: Mi = A + B̄ + C

Esta representación canónica es fundamental porque permite convertir cualquier tabla de verdad en una expresión algebraica estándar, facilitando la simplificación y el diseño de circuitos digitales.

5. Ejemplo: Tabla de verdad con minitérminos y maxitérminos

Analicemos una función práctica F(A, B, C) para comprender cómo se obtienen las formas canónicas:

A B C F Índice Tipo
0 0 0 0 0 M₀
0 0 1 1 1 m₁
0 1 0 0 2 M₂
0 1 1 1 3 m₃
1 0 0 1 4 m₄
1 0 1 0 5 M₅
1 1 0 1 6 m₆
1 1 1 1 7 m₇

🔹 Forma canónica de suma de minitérminos (SOP):

F = Σm(1, 3, 4, 6, 7)

F = Ā·B̄·C + Ā·B·C + A·B̄·C̄ + A·B·C̄ + A·B·C

Esta expresión suma todos los minitérminos donde la función F vale 1. Cada término representa exactamente una fila de la tabla de verdad.

🔹 Forma canónica de producto de maxitérminos (POS):

F = ΠM(0, 2, 5)

F = (A + B + C)·(A + B̄ + C)·(Ā + B + C̄)

Esta expresión multiplica todos los maxitérminos donde la función F vale 0. Es la forma dual de la representación SOP.

6. ¿Cómo usar minitérminos y maxitérminos?

Las formas canónicas son herramientas fundamentales en el diseño digital porque proporcionan un punto de partida estándar para la simplificación de circuitos. Son especialmente útiles para expresar funciones lógicas en formas estándar que facilitan la simplificación, el diseño de circuitos y la implementación en compuertas lógicas.

En circuitos digitales, la forma SOP se traduce directamente en una arquitectura de dos niveles donde las compuertas AND procesan las variables de entrada para generar los minitérminos, y una compuerta OR final suma todos estos productos para obtener la función completa.

7. Sugerencia Visual: Circuito Lógico de un SOP

Para implementar la función F = Ā·B̄·C + Ā·B·C + A·B̄·C̄ + A·B·C̄ + A·B·C, el circuito tendría:

  • Entradas: A, B, C y sus complementos Ā, B̄, C̄
  • Primer nivel: 5 compuertas AND de 3 entradas cada una, generando los minitérminos
  • Segundo nivel: 1 compuerta OR de 5 entradas que suma todos los productos
  • Salida: La función F resultante

Este diseño de dos niveles es la base para muchas implementaciones en FPGAs y circuitos integrados digitales.

8. Actividades Recomendadas

Para consolidar tu comprensión del álgebra de Boole y las formas canónicas, te recomiendo realizar las siguientes actividades prácticas:

  • Realizar una tabla de verdad completa con 3 variables (A, B, C) para una función de tu elección
  • Identificar y obtener todos los minitérminos y maxitérminos de esa función
  • Representar la función tanto como SOP (Suma de Productos) como POS (Producto de Sumas)
  • Verificar matemáticamente con tabla de verdad que ambas representaciones son equivalentes
  • Implementar el diseño en un simulador digital como Logisim, Proteus o Tinkercad para validar prácticamente tu trabajo
  • Intentar simplificar las expresiones obtenidas usando los postulados del álgebra de Boole
  • Comparar el número de compuertas necesarias antes y después de la simplificación

Estas actividades te ayudarán a desarrollar una comprensión sólida de los conceptos teóricos y su aplicación práctica en el diseño de circuitos digitales.
